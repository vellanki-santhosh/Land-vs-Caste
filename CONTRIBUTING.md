# Contributing to Land vs Caste

Thank you for your interest in contributing. This repository is a collaborative research resource on land ownership and caste in India. Contributions of verified data, factual corrections, regional case studies, and new analytical sections are welcome.

---

## What We Welcome

- **Verified data additions** — CSV rows or new datasets sourced from government surveys (NSSO, Agricultural Census, NFHS), peer-reviewed research, or credible NGO reports.
- **Factual corrections** — If you find an error in any document, please open an issue or pull request with the corrected information and the source.
- **Regional case studies** — Detailed documentation of land-caste dynamics in specific districts, states, or communities, with primary sources.
- **Legal updates** — New court decisions, legislation, or policy developments relevant to Dalit/Adivasi land rights.
- **Translation** — Translations of key sections into Indian languages are encouraged.

---

## How to Contribute

1. **Fork** the repository and create a new branch from `main`.
2. Make your changes. See style guidelines below.
3. Open a **Pull Request** with:
   - A clear description of what you changed and why.
   - Citations for any factual claims (see citation format below).
4. A maintainer will review your PR and may request clarification or additional sources.

---

## Citation Format

Use the following format for inline citations in Markdown documents:

```
(Author Last Name, Year; Author Last Name, Year)
```

For government data:

```
(Agency Name, Report Title, Year)
```

Add full references to the **Further Reading** section at the end of each document.

---

## Data Files

- All CSV files must include a `source` column and a `survey_year` or `year` column.
- Do not alter the CSV header row format without updating the corresponding documentation.
- Data from unpublished or unverified sources will not be accepted.

---

## Style Guidelines

- Use clear, neutral language. Avoid editorialising; let data and cited analysis make the argument.
- Headings use sentence case (e.g., "The reform that wasn't", not "The Reform That Wasn't").
- Use Markdown tables for structured comparisons.
- Keep documents factual and grounded; speculation should be clearly labelled as such.
